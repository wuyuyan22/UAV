# 伴飞程序（swarm_node）运行文档

本文档说明在 **鲁班猫机载 Linux + ArduPilot 飞控** 环境下，如何通过命令行启动 **UAV Swarm Controller**（可执行文件 `swarm_node`），完成长机/僚机伴飞、5G+VPN 地面站链路与机间 UDP 群控联调。内容基于仓库源码 `src/main.cpp` 与实机终端日志（`lubancat:~/UAV/build`）。

---

## 1. 架构与角色

```
┌──────────────┐  串口 /dev/ttyS0@57600   ┌─────────────┐
│  鲁班猫伴飞   │ ◄──────────────────────► │ ArduPilot   │
│  swarm_node  │                          │  飞控 FC     │
└──────┬───────┘                          └─────────────┘
       │
       ├── GCS 链路 (--gcs-link ip) ──UDP──► 10.8.0.100:14550  (Mission Planner 等)
       │
       └── 机间链路 (UDP :19870) ◄────────► 长机 ↔ 僚机 (VPN 私网，如 10.8.0.x)
```

| 角色 | 命令行 `role` | 含义 |
|------|---------------|------|
| 长机 | `1` | 汇聚 GCS 指令、管理 `--followers` 列表、监听机间 UDP |
| 僚机 | `2` | 通过 `--leader-ip` 指向长机 VPN IP，上报/接收群控数据 |

**启动顺序建议**：先起长机，再起僚机；僚机日志出现 `[udp-link] peer sysid=N -> <长机IP>:19870` 表示机间链路已建立。

---

## 2. 构建

在机载板（或交叉编译环境）执行：

```bash
cd /root/UAV
mkdir -p build && cd build
cmake .. && make -j4
```

产物路径：**`/root/UAV/build/swarm_node`**（下文命令均假设在此目录执行）。

查看完整参数说明：

```bash
./swarm_node
# 或
./swarm_node --help   # 参数不足时 main 会打印 Usage
```

---

## 3. 命令行格式

```text
./swarm_node <sysid> <role> [options...]
```

| 位置参数 | 取值 | 说明 |
|----------|------|------|
| `sysid` | 1–255 | MAVLink 系统 ID，须与飞控 `SYSID_THISMAV`、地面站显示一致 |
| `role` | `1` = LEADER，`2` = FOLLOWER | 决定长机/僚机逻辑分支 |

其余均为可选 `--` 开关；未指定时使用 `main.cpp` 内默认值（见 §5）。

## 4. 本环境实测启动命令

以下命令来自当前联调终端（WireGuard 网段 `10.8.0.0/24`：地面站 `10.8.0.100`，长机 `10.8.0.3`，僚机 sysid=2）。**部署到其它飞机时请按 VPN IP 表替换 IP。**

### 4.1 僚机（sysid=2）

```bash
cd /root/UAV/build

./swarm_node 2 2 \
  --serial /dev/ttyS0 --baud 57600 \
  --type copter \
  --leader-ip 10.8.0.3 \
  --link-port 19870 \
  --gcs-link ip \
  --gcs-udp-target 10.8.0.100:14550 \
  --no-follower-synthetic-telem-gcs \
  --hz 10
```

**预期关键日志：**

| 日志片段 | 含义 |
|----------|------|
| `role : FOLLOWER` | 角色正确 |
| `[gcs_ip] bind 0.0.0.0:0 -> target 10.8.0.100:14550` | GCS 走 5G/VPN UDP，本地任意口出站 |
| `[udp] bound 0.0.0.0:19870` | 机间 UDP 监听 |
| `[udp-link] peer sysid=2 -> 10.8.0.3:19870` | 已发现长机地址 |
| `[swarm] init ok, sysid=2 role=FOLLOWER` | 初始化成功 |

### 4.2 长机（sysid=1）

```bash
cd /root/UAV/build

./swarm_node 1 1 \
  --serial /dev/ttyS0 --baud 57600 \
  --type copter \
  --followers 2,3 \
  --link-port 19870 \
  --gcs-link ip \
  --gcs-udp-target 10.8.0.100:14550 \
  --gcs-udp-bind 10.8.0.3:14551 \
  --no-follower-synthetic-telem-gcs \
  --hz 10
```

**预期关键日志：**

| 日志片段 | 含义 |
|----------|------|
| `role : LEADER`，`followers : 2,3` | 长机管辖僚机列表 |
| `[gcs_ip] bind 10.8.0.3:14551 -> target 10.8.0.100:14550` | **显式绑定 wg0 地址**，避免源地址走蜂窝 wwan0 |
| `[swarm] GCS link enabled: type=0, data_stream=4Hz` | 向 GCS 聚合遥测（约 4Hz） |
| `[gcs_tx] ... sent_pkt=... sent_bytes=...` | 上行令牌桶在发数；`drop_*=0` 表示未丢包 |
| `[swarm] leader recv GCS COMMAND_LONG: cmd=...` | 地面站指令已进入长机 |

正常退出：`Ctrl+C` → `[main] signal received, stopping...` → `[main] exited`。

## 5. 常用参数说明

### 5.1 飞控链路

| 参数 | 默认 | 说明 |
|------|------|------|
| `--serial <dev>` | `/dev/ttyS0` | 鲁班猫 4：40pin 8/10 → UART0_M2；其它板型用 `ls -l /dev/ttyS*` 确认 |
| `--baud <rate>` | `57600` | 须与飞控 `SERIALx_BAUD` 一致 |
| `--udp <ip:port>` | — | SITL/仿真时替代串口 |
| `--type copter\|plane` | `copter` | 多旋翼 / 固定翼 |

### 5.2 地面站（GCS）链路 — 5G + VPN 主路径

| 参数 | 默认 | 说明 |
|------|------|------|
| `--gcs-link ip\|serial\|p900\|none` | `ip` | 本联调使用 `ip`（UDP 经 VPN） |
| `--gcs-udp-target <ip:port>` | `10.8.0.100:14550` | Mission Planner 等监听口 |
| `--gcs-udp-bind <ip:port>` | `0.0.0.0:0` | **长机建议**设为 `10.8.0.x:14551`，绑定 VPN 网卡 |
| `--gcs-udp-tx-kbps <n>` | `384` | 上行带宽令牌桶上限 |
| `--gcs-heartbeat-hz <n>` | `1` | 向 GCS 发 HEARTBEAT 保 NAT |
| `--no-follower-synthetic-telem-gcs` | （`ip` 模式默认关闭合成遥测） | 不在长机侧“伪造”僚机 MAVLink 流；各机各自上报 |

### 5.3 机间群控链路

| 参数 | 默认 | 说明 |
|------|------|------|
| `--link-port <p>` | `19870` | 长机/僚机均监听；僚机发包目标为 `--leader-ip:port` |
| `--leader-ip <ip>` | — | **仅僚机必填**，长机 VPN IP |
| `--followers <a,b>` | 长机默认 `2,3` | **仅长机**，声明管辖的僚机 sysid |

### 5.4 控制与健壮性

| 参数 | 默认 | 说明 |
|------|------|------|
| `--hz <n>` | `10` | 控制环频率（≥2） |
| `--health-disable` | 监视默认开启 | 关闭 5G 链路健康检测 |
| `--health-warn-ms` / `--health-lost-ms` | `3000` / `8000` | 收包静默 → 降级 / 丢失阈值 |
| `--p900-uart9` / `--p900-uart9-fallback` | 关 | P900 串口兜底（弱信号外场） |
| `--sim` | 关 | 仿真模式 |

## 6. 网络与端口一览

| 用途 | 协议 | 地址（示例） | 节点 |
|------|------|--------------|------|
| GCS 下行目标 | UDP | `10.8.0.100:14550` | 地面站 / MP |
| GCS 长机源绑定 | UDP | `10.8.0.3:14551` | 长机 wg0（可选但推荐） |
| 机间群控 | UDP | `0.0.0.0:19870` 监听 | 长机 + 僚机 |
| 机间对端 | UDP | `10.8.0.3:19870` | 僚机 `--leader-ip` |

联调前建议：

```bash
wg show
ping -c3 10.8.0.100    # GCS
ping -c3 10.8.0.3      # 长机（僚机侧）
ss -upn | grep -E '14550|14551|19870'
```

## 7. systemd 开机自启（可选）

仓库提供单元模板，安装后由环境文件区分长机/僚机：

| 文件 | 作用 |
|------|------|
| `systemd/swarm-controller@.service` | `swarm-controller@<sysid>` 实例 |
| `systemd/etc-default-swarm-1.env` | 长机 `EXTRA_ARGS`（含 `--followers`、`--gcs-udp-bind`） |
| `systemd/README_LEADER_VPN.md` | VPN、QoS、启动链说明 |

长机环境文件示例（IP 与 §4 不同处请改 `EXTRA_ARGS`）：

```bash
EXTRA_ARGS=--followers 2,3 --gcs-udp-bind 10.8.0.1:14551
```

## 8. 启动自检清单

1. **串口**：`[serial] /dev/ttyS0 opened @ 57600 baud`，无 `init failed`。
2. **角色横幅**：`sysid` / `role` / `FC link` / `GCS link` 与命令一致。
3. **机间**：僚机出现 `[udp-link] peer ... -> <长机IP>:19870`。
4. **GCS 上行**：长机运行一段时间后 `[gcs_tx]` 中 `sent_pkt`、`sent_bytes` 递增，`drop_h=0 drop_n=0`。
5. **5G 健康**：`5G health : enabled`，日志 `warn=3000ms lost=8000ms`。

## 9. 常见问题

### 9.1 地面站连上但 cmd=512 返回 FAIL(4)

日志示例：

```text
[swarm] leader recv GCS COMMAND_LONG: cmd=512 src=255/190 target=0/0
[vehicle 1] CMD_ACK cmd=512 result=FAIL(4)
```

`512` = `MAV_CMD_REQUEST_MESSAGE`（Mission Planner 拉取单条消息）。伴飞节点转发给飞控后，若飞控不支持该请求会 **FAIL**，多数情况下 **不影响基本遥测与指令**，可忽略或关闭 MP 中对应“单次请求”类功能。

### 9.2 僚机无 `[udp-link] peer`

- 确认长机已启动且 `--link-port` 一致。
- 确认 `--leader-ip` 为长机当前 **VPN IP**（非蜂窝公网 IP）。
- 检查防火墙/NAT 是否拦截 UDP 19870。

### 9.3 GCS 无数据、`sent_pkt=0`

- 长机是否配置 `--gcs-udp-bind` 为 wg0 地址。
- `ping 10.8.0.100` 是否通；WireGuard 是否 `active`。
- 飞控串口是否有心跳（伴飞需先收到 FC 遥测才会聚合上报）。

### 9.4 串口权限

```bash
sudo usermod -aG dialout $USER
# 或临时
sudo chmod 666 /dev/ttyS0
```
