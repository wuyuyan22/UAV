# 声网（Agora）直播推流运行文档

本文档说明在 **鲁班猫 4（RK3588）** 机载 Linux 上，使用 **GStreamer + gst-agora 插件（agorasink）** 完成摄像头采集、H.264 编码、声网 RTC 频道推流，以及地面 **Web 观众端** 拉流观看的完整流程。与 [RTSP推流运行文档.md](./RTSP推流运行文档.md) 互补：RTSP 用于本机/局域网环回，本文档用于 **公网/声网频道** 图传联调。

---

## 1. 架构概览

```
┌──────────────┐  Annex-B   ┌───────────┐  完整 AU   ┌─────────────┐   Agora RTC   ┌──────────────────┐
│  GStreamer   │ ─────────► │ h264parse │ ─────────► │  agorasink  │ ────────────► │ 声网 SD-RTN 云端  │
│ v4l2src      │            │(硬编必选) │            │ (gst-agora) │               └────────┬─────────┘
│ mpph264enc   │            └───────────┘            │ userid=101  │                        │
│ 或 x264enc   │                                    └─────────────┘                        ▼
└──────────────┘                                                              ┌──────────────────┐
      机载推流端 (Broadcaster)                                                 │ agora-viewer.html │
                                                                               │ Web SDK 观众端     │
                                                                               └──────────────────┘
```

| 组件 | 路径/说明 | 作用 |
|------|-----------|------|
| gst-agora 源码 | `/root/agora/agora/agora-gstreamer-main/gst-agora` | 含 `agorasink` 插件与 `libagorac` |
| 运行时插件 | `/usr/local/lib/aarch64-linux-gnu/gstreamer-1.0/libgstagorasink.so` | `install` 后由 `GST_PLUGIN_PATH` 加载 |
| 编译产物 | `.../gst-agora/build/plugin-src/libgstagorasink.so` | 改源码后 `ninja`，可临时覆盖 `GST_PLUGIN_PATH` 指向此目录 |
| 观众页 | `/root/UAV/agora-viewer.html` | 浏览器加入同一频道观看 |
| 摄像头 | `/dev/video11`（`rkisp_mainpath`） | LubanCat-4 上 ISP 输出节点；`/dev/video0` 为 rkcif 原始 Bayer，勿用于 UYVY 推流 |

**频道参数（与观众页一致）**

| 参数 | 示例值 | 说明 |
|------|--------|------|
| App ID | `9e6e7036db144983aa62a1548a0367a9` | 声网项目 App ID |
| Channel | `test` | 频道名 |
| User ID | `101` | 机载推流端 UID（字符串） |

正式环境若启用 **Token 鉴权**，需在推流端与 `agora-viewer.html` 中同时配置 Token（当前为无 Token 测试模式）。

## 2. 环境变量（已写入 ~/.bashrc）

插件安装在 `/usr/local/lib/...`，而 GStreamer 默认只搜索 `/usr/lib/aarch64-linux-gnu/gstreamer-1.0/`，**不设置环境变量会出现「无组件 agorasink」**。

已在 `~/.bashrc` 末尾配置：

```bash
export GST_PLUGIN_PATH=/usr/local/lib/aarch64-linux-gnu/gstreamer-1.0
export LD_LIBRARY_PATH=/usr/local/lib:${LD_LIBRARY_PATH}
```

**生效方式**（任选其一）：

```bash
source ~/.bashrc    # 当前终端立即生效
# 或新开一个终端
```

下文推流命令**不再重复** `export`；若在未加载 bashrc 的环境（如 systemd、cron）中运行，需自行带上上述两行。

### 2.1 前置检查

```bash
# GStreamer 与 RK 硬编插件
gst-inspect-1.0 v4l2src mpph264enc videoconvert 2>/dev/null | grep -E "^(Plugin|Factory)"

# agorasink（应能看到 Factory Details）
gst-inspect-1.0 agorasink

# 摄像头
v4l2-ctl --list-devices
v4l2-ctl --device=/dev/video11 --list-formats-ext 2>/dev/null | head -20
```

**MPP 探测告警可忽略**：执行 `gst-inspect-1.0 mpph264enc` 或启动管道时可能出现 `unable to create enc vp8`、`JPEG encoder device` 等日志，为部分 RK 平台不支持 VP8/JPEG 硬编的副作用，**不影响 H.264 推流**。

## 3. 编译与加载 agorasink

### 3.1 全量编译（首次或升级 SDK）

```bash
cd /root/agora/agora/agora-gstreamer-main/build
chmod +x build_all_aarch64_4.2.32.sh agora/libagorac/install.sh gst-agora/install
./build_all_aarch64_4.2.32.sh
```

脚本会将 `libgstagorasink.so` 安装到 `/usr/local/lib/aarch64-linux-gnu/gstreamer-1.0/`。

### 3.2 增量编译（仅修改插件源码后）

```bash
cd /root/agora/agora/agora-gstreamer-main/gst-agora/build
ninja
sudo cp plugin-src/libgstagorasink.so \
  /usr/local/lib/aarch64-linux-gnu/gstreamer-1.0/
```

若无 `build` 目录，需在 `gst-agora` 下用 meson 配置一次（按项目 README）。

**临时使用 build 目录插件**（未 `cp` 到 `/usr/local` 时）：

```bash
export GST_PLUGIN_PATH=/root/agora/agora/agora-gstreamer-main/gst-agora/build/plugin-src
```

### 3.3 硬编推流要点（机载必读）

**`mpph264enc` 与 Web 观众端的兼容关键：编码器与 `agorasink` 之间必须加 `h264parse`。**

| 现象（不加 h264parse） | 原因 | 处理 |
|------------------------|------|------|
| 首帧能解、P 帧立刻失败 | 硬编按 **单 NAL 逐 buffer** 输出，Web SDK 无法稳定解码 P 帧参考链 | 加 `h264parse` 合并为完整 **Access Unit** |
| 反复 `INTRA_REQUEST` / 高卡顿率 | 观众端解码失败后持续索要 I 帧 | 同上 |
| 软编 `x264enc` 正常、硬编异常 | 软编天然输出完整 AU；硬编需 `h264parse` 对齐 | §4 管道统一加 `h264parse config-interval=1` |

推荐写法（硬编）：

```bash
mpph264enc profile=baseline header-mode=each-idr gop=30 bps=2000000 ! \
h264parse config-interval=1 ! \
agorasink ...
```

| 参数 | 说明 |
|------|------|
| `h264parse` | 将硬编散 NAL 合并为按帧 AU，**Web 端 P 帧可解码**（已实测 30fps / 0% 卡顿） |
| `config-interval=1` | 周期性在码流中插入 SPS/PPS，观众中途进频道更稳 |
| `header-mode=each-idr` | 与 `h264parse` 配合，每个 IDR 仍带参数集 |

`agorasink` 插件另有 **退出时释放声网连接**（EOS / `NULL` 时 `agoraio_disconnect()`），避免 UID 占线；改插件源码后需 `ninja` 并安装。

## 4. 推流管道（RK 硬件编码）

**通用约定**

| 规则 | 说明 |
|------|------|
| **`h264parse`（硬编必选）** | 放在 `mpph264enc` 与 `agorasink` 之间，`config-interval=1`；**不可省略**（见 §3.3） |
| **`gop`** | **等于目标帧率**（30fps → `gop=30`，60fps → `gop=60`），约 1 秒一个 IDR |
| **`bps`** | 帧率翻倍时码率约 **×1.5～2**；分辨率升高再叠加（见下表） |
| **caps `framerate`** | `video/x-raw,...,framerate=N/1` 与 `gop`、`agorasink` 送流 fps 一致 |
| **`agorasink dfps`** | `0`（默认）= 从 `video/x-h264` caps 自动读帧率；也可显式 `dfps=60` |
| 勿加 `-e` | 直播推流见 §9.6 |
| 勿加 `video/x-h264,profile=baseline` caps | 易与硬编协商失败，见 §4 末说明 |

**帧率 / GOP / 码率对齐示例**

| 目标 fps | caps `framerate` | `gop` | 640×360 `bps` | 1280×720 `bps` | 1920×1080 `bps` |
|----------|------------------|-------|---------------|----------------|-----------------|
| 15 | `15/1` | 15 | 1.2M | 2.0M | 3.5M |
| 30 | `30/1` | 30 | 2.0M | 3.5M | 5.0M |
| 60 | `60/1` | 60 | 3.5M | 6.0M | 8.0M+ |

**帧率档位**

| 档位 | 小节 | 分辨率@fps | 码率参考 |
|------|------|------------|----------|
| 省流稳定 | §4.1 | 640×360@15 | 1.2 Mbps |
| **高帧率直播** | **§4.2** | **640×360@30** | 2.0 Mbps |
| 高清高帧率 | §4.3 | 1280×720@30 | 3.5 Mbps |
| 全高清高帧率 | §4.4 | 1920×1080@30 | 5～6 Mbps |
| 高帧率 60fps | §4.5 | 640×360 或 720p@60 | 见上表 |

将下文 `appid`、`channel`、`userid` 替换为实际值；与 `agora-viewer.html` 一致。  
`agorasink` 初始化时会打印 `dfps=N`，应与 caps 帧率一致。

### 4.1 标清稳定（640×360@15fps）

已验证 Web 端可正常解码，适合 **5G 带宽紧张** 或首调链路。

```bash
gst-launch-1.0 \
  v4l2src device=/dev/video11 ! \
  video/x-raw,format=UYVY,width=640,height=360,framerate=15/1 ! \
  videoconvert ! video/x-raw,format=NV12 ! \
  mpph264enc profile=baseline header-mode=each-idr gop=15 bps=1200000 ! \
  h264parse config-interval=1 ! \
  agorasink \
    appid=9e6e7036db144983aa62a1548a0367a9 \
    channel=test \
    userid=101 \
    verbose=true
```

| 参数 | 说明 |
|------|------|
| `profile=baseline` | 与 Web SDK `codec: 'h264'` 兼容性好 |
| `header-mode=each-idr` | 每个 IDR 附带 SPS/PPS |
| `gop=15` | 15fps 下约 1 秒一个 I 帧 |
| `bps=1200000` | 约 1.2 Mbps |
| `h264parse config-interval=1` | 硬编合并 AU，Web 端可稳定解码 P 帧 |

### 4.2 高帧率直播（640×360@30fps，推荐）

在 RK3588 上提高 **Out video fps**，浏览器「接收 fps」更流畅；上行建议 ≥ **3 Mbps**。本档位为 **量产推荐**，Web 端实测可达 **~30fps、0% 卡顿**。

```bash
gst-launch-1.0 \
  v4l2src device=/dev/video11 ! \
  video/x-raw,format=UYVY,width=640,height=360,framerate=30/1 ! \
  videoconvert ! video/x-raw,format=NV12 ! \
  mpph264enc profile=baseline header-mode=each-idr gop=30 bps=2000000 ! \
  h264parse config-interval=1 ! \
  agorasink \
    appid=9e6e7036db144983aa62a1548a0367a9 \
    channel=test \
    userid=101 \
    verbose=true
```

成功时终端 `Out video fps` 宜在 **20～30** 区间（非 §4.1 的 ~10）。

### 4.3 高清高帧率（1280×720@30fps）

需要更清晰画面且链路 ≥ **5 Mbps** 时使用：

```bash
gst-launch-1.0 \
  v4l2src device=/dev/video11 ! \
  video/x-raw,format=UYVY,width=1280,height=720,framerate=30/1 ! \
  videoconvert ! video/x-raw,format=NV12 ! \
  mpph264enc profile=baseline header-mode=each-idr gop=30 bps=3500000 ! \
  h264parse config-interval=1 ! \
  agorasink \
    appid=9e6e7036db144983aa62a1548a0367a9 \
    channel=test \
    userid=101 \
    verbose=true
```

### 4.4 全高清高帧率（1920×1080@30fps）

带宽充足（建议 ≥ **8 Mbps**）的直播场景；若出现绿屏/花屏，先降回 §4.2 或 §4.3。

```bash
gst-launch-1.0 \
  v4l2src device=/dev/video11 ! \
  video/x-raw,format=UYVY,width=1920,height=1080,framerate=30/1 ! \
  videoconvert ! video/x-raw,format=NV12 ! \
  mpph264enc profile=baseline header-mode=each-idr gop=30 bps=5000000 ! \
  h264parse config-interval=1 ! \
  agorasink \
    appid=9e6e7036db144983aa62a1548a0367a9 \
    channel=test \
    userid=101 \
    verbose=true
```

### 4.5 高帧率 60fps（640×360 或 720p，实验）

`agorasink` 已从 `video/x-h264` caps **自动同步送流 fps**（`dfps=0` 默认）。须保证 **caps `framerate`、`gop`、`bps` 三者对齐**。

```bash
# 640×360@60fps（RK3588 较易稳定）
gst-launch-1.0 \
  v4l2src device=/dev/video11 ! \
  queue max-size-buffers=2 leaky=downstream ! \
  video/x-raw,format=UYVY,width=640,height=360,framerate=60/1 ! \
  videoconvert ! video/x-raw,format=NV12 ! \
  mpph264enc profile=baseline header-mode=each-idr gop=60 bps=3500000 ! \
  h264parse config-interval=1 ! \
  agorasink \
    appid=9e6e7036db144983aa62a1548a0367a9 \
    channel=test \
    userid=101 \
    verbose=false
```

| 参数 | 说明 |
|------|------|
| `framerate=60/1` + `gop=60` | 采集与 I 帧间隔对齐 |
| `bps=3500000` | 60fps 下约 3.5 Mbps |
| `dfps` 省略 | 自动从 caps 读 **60**；初始化日志应显示 `dfps=60` |
| `dfps=60` | 显式指定（caps 无 framerate 时用） |
| `Out video fps` | 目标 **55～60**；若仍 ~30，检查是否未重编插件 |

1080p@60 硬编在 RK3588 上通常难以稳定，不建议量产使用。

**不要**在硬编链路上随意加 `! video/x-h264,profile=baseline !` caps 过滤；**必须**保留 `h264parse config-interval=1`（见 §3.3）。

## 5. 备选：软件编码（链路验证）

软编 `x264enc` 天然输出完整 Access Unit，**无需** `h264parse`，适合快速验证声网频道与 Web 观众页是否正常：

```bash
gst-launch-1.0 \
  v4l2src device=/dev/video11 ! \
  video/x-raw,format=UYVY,width=640,height=360,framerate=30/1 ! \
  videoconvert ! video/x-raw,format=I420 ! \
  x264enc key-int-max=30 tune=zerolatency bitrate=2000 speed-preset=ultrafast ! \
  agorasink \
    appid=9e6e7036db144983aa62a1548a0367a9 \
    channel=test \
    userid=101 \
    verbose=false
```

| 说明 | 建议 |
|------|------|
| `key-int-max=30` | 30fps 下约 1 秒一个 I 帧 |
| `verbose=false` | 软编时关闭逐帧打印，减轻 CPU 负担 |
| 与硬编对比 | 软编正常而硬编卡顿时，检查 §4 是否遗漏 `h264parse` |
| 量产 | 数传场景请用 §4 硬编 + `h264parse` |

## 6. 地面 Web 观众端

### 6.1 打开页面

将 `/root/UAV/agora-viewer.html` 拷到可访问的 PC（或直接在本机用浏览器打开），确认其中：

- `APP_ID`、`CHANNEL` 与推流命令一致；
- `codec: 'h264'`（已配置）；
- 角色为 `audience`，`level: 2`（超低延迟观众，抖动缓冲约 0.5s；需硬编管道含 `h264parse`）。

### 6.2 观看步骤

1. **先**在机载启动 §4.1 或 §4.2 推流管道，等待日志出现 `agora has been successfuly initialized`。
2. 浏览器打开 `agora-viewer.html`，点击 **「加入频道」**。
3. 日志应出现：`发现远端流: user=101` → `已订阅: video` → `视频播放中`。
4. 页面底部应显示 **渲染 ~30 fps、SDK 接收 ~30 fps、卡顿 0%**（硬编需 §4 含 `h264parse`）。

### 6.3 浏览器控制台正常标志

- `subscribe success user 101, mediaType: video`
- **无** `RECV_VIDEO_DECODE_FAILED`（错误码 1005）

## 7. 推流成功标志（机载终端）

```text
agora has been successfuly initialized
*********first key frame sent (NAL detected) **********
agorasink: received xxxx bytes (send xxxx)!
agorasink: is key frame: 1
Out video fps: 10
```

| 日志 | 含义 |
|------|------|
| `Out video fps: 10` | 多为 §4.1（15fps 档）正常发送速率 |
| `Out video fps: 20～30` | §4.2～4.4 高帧率直播正常 |
| `In video fps: 0` | 本机为 **仅发送（sendOnly）**，未订阅他人视频，**恒为 0 属正常** |
| `is key frame: 1` | 检测到 IDR/SPS 关键帧 |
| 大 I 帧 `verbose` 首 NAL 为 `103/104`（SPS/PPS）再为 `101`（IDR） | 参数集齐全，Web 可解码 |

推流前确认 `/dev/video11` 未被占用：

```bash
sudo fuser -v /dev/video11

ps aux | grep -E 'gst-launch|ffmpeg|ffplay|mediamtx'
```

## 9. 常见问题

### 9.0 `警告: 错误管道: 无组件“agorasink”`

| 原因 | 处理 |
|------|------|
| 未加载 `~/.bashrc` 中的环境变量 | `source ~/.bashrc` 或新开终端 |
| 插件未安装 | 执行 §3.1 全量编译，或确认 `/usr/local/lib/aarch64-linux-gnu/gstreamer-1.0/libgstagorasink.so` 存在 |
| 依赖库缺失 | 确认 `LD_LIBRARY_PATH` 含 `/usr/local/lib`，执行 `gst-inspect-1.0 agorasink` 查看加载错误 |

### 9.1 终端无 `agorasink: received`

| 可能原因 | 处理 |
|----------|------|
| 未设置环境变量 | 见 §2、§9.0 |
| 摄像头被占用 | 见 §8 |
| 启动后过早 Ctrl+C | 至少等待 10～15 秒 |
| 管道元素链接失败 | 去掉 `-e` 后重试，执行 `gst-inspect-1.0 agorasink` |

### 9.2 浏览器 `RECV_VIDEO_DECODE_FAILED`（1005）

| 可能原因 | 处理 |
|----------|------|
| **硬编未加 `h264parse`** | 在 `mpph264enc` 与 `agorasink` 之间加 `h264parse config-interval=1`（§3.3、§4） |
| IDR 前无 SPS/PPS | 使用 `header-mode=each-idr` + `h264parse config-interval=1` |
| 观众先进频道、错过首帧 | 缩短 `gop`；重新加入频道或重启推流 |
| 码流为 High Profile | 设置 `mpph264enc profile=baseline` |

### 9.3 反复 `Agora sdk requested an iframe`

推流过程中出现：SDK / 观众认为 **P 帧不可解码**，索要带参数集的 I 帧。

| 可能原因 | 处理 |
|----------|------|
| 硬编单 NAL 直送、无 `h264parse` | **首选**：按 §4 加上 `h264parse config-interval=1` |
| 软编正常、硬编异常 | 对比 §5 与 §4.2，确认硬编管道含 `h264parse` |
| 仅退出推流时出现 | 编码已停，属正常现象 |

**退出管道时**大量 iframe 请求属正常现象（编码已停）。

### 9.4 `In video fps: 0` 但 Out 正常

推流端 **不表示** 观众失败；仅表示机载未接收频道内其他用户的视频。以 Web 端 `接收 fps` 与是否 1005 为准。

### 9.5 MPP 初始化报错

```text
mpp: unable to create enc vp8 ...
mpp: error found on mpp initialization
```

多为插件探测，后续出现 `MPP_ENC_SET_RC_CFG` 即表示 **H.264 编码已工作**。

### 9.6 Ctrl+C 后卡在「等待 EOS 信号...」

**原因**：命令行带了 **`-e`**（`--eos-on-shutdown`）。中断后 `gst-launch` 会向管道发 EOS 并**一直等到收齐**；`v4l2src` 是直播源，往往**不会正常结束**，于终端停在：

```text
允许关闭时发出 EOS 信号 —— 强制对管道发出 EOS 信号
等待 EOS 信号...
```

旧版 `agorasink` 未在退出时 `disconnect` 声网，会加重占线；新版插件会在 EOS / `READY→NULL` 时打印 `Agoraio: disconnected` 并释放资源，但**仍建议直播推流不要用 `-e`**。

**推荐退出方式**：

1. **一次** `Ctrl+C`（不要用 `-e` 启动时通常几秒内可退出）。
2. 若仍卡住：再按 `Ctrl+C`，或另开终端：

```bash
sudo pkill -f gst-launch-1.0
```

3. 确认摄像头释放：`sudo fuser -v /dev/video11`（无输出即可再推）。

**多次推流前**：先 `pkill`，再启动 §4 管道，避免 UID 101 与 `/dev/video11` 冲突。

### 9.7 画面发绿、0 fps 或高帧率仍卡顿

| 情况 | 处理 |
|------|------|
| 硬编首帧能解、随后卡顿 / 高卡顿率 | 检查是否遗漏 `h264parse config-interval=1`（§3.3） |
| 1080p + 低码率 | `bps=1200000` 不足，改用 §4.4 的 `bps=5000000` 或降 §4.2 |
| 高帧率但 Out fps 仍 ~10 | caps 未写 `framerate=30/1`，或 `gop`/`bps` 仍按 15fps 配置 |
| 改分辨率/帧率后异常 | `sudo pkill -f gst-launch-1.0` 后重启管道 |
| Web 接收 fps 低 | 先确认 `h264parse`；再升 `bps` 或改 §4.2 640×360@30 |

## 11. 快速命令备忘

```bash
# 环境（已写入 ~/.bashrc，新开终端自动生效）
source ~/.bashrc
gst-inspect-1.0 agorasink

# 4.1 标清 15fps（省带宽）
gst-launch-1.0 v4l2src device=/dev/video11 ! \
  video/x-raw,format=UYVY,width=640,height=360,framerate=15/1 ! videoconvert ! \
  video/x-raw,format=NV12 ! mpph264enc profile=baseline header-mode=each-idr \
  gop=15 bps=1200000 ! h264parse config-interval=1 ! \
  agorasink appid=<APP_ID> channel=<CH> userid=101 verbose=true

# 4.2 高帧率直播 30fps（推荐，硬编必加 h264parse）
gst-launch-1.0 v4l2src device=/dev/video11 ! \
  video/x-raw,format=UYVY,width=640,height=360,framerate=30/1 ! videoconvert ! \
  video/x-raw,format=NV12 ! mpph264enc profile=baseline header-mode=each-idr \
  gop=30 bps=2000000 ! h264parse config-interval=1 ! \
  agorasink appid=<APP_ID> channel=<CH> userid=101 verbose=true

# 4.3 高清高帧率 720p@30
gst-launch-1.0 v4l2src device=/dev/video11 ! \
  video/x-raw,format=UYVY,width=1280,height=720,framerate=30/1 ! videoconvert ! \
  video/x-raw,format=NV12 ! mpph264enc profile=baseline header-mode=each-idr \
  gop=30 bps=3500000 ! h264parse config-interval=1 ! \
  agorasink appid=<APP_ID> channel=<CH> userid=101 verbose=true

# 释放摄像头
sudo pkill -f gst-launch-1.0
```
