# 无人机群控机间避障设计文档

本文档定义 **UAV_Swarm_APM**（`swarm_node`）在多机编队与独立任务场景下的**机间碰撞检测与避障**工程方案。内容基于当前仓库源码（`SwarmController`、`FormationCalculator`、`MissionSplitter`、`InterUavLink`），面向后续模块开发与 SITL/实飞验证。

**文档版本**：v1.0  
**适用代码基线**：`/root/UAV` 当前 main 分支  
**关联文档**：
- `doc/伴飞程序运行文档.md` — 部署与联调
- `MISSION_SWARM_IMPLEMENTATION_CN.md` — 群控任务与状态机
- `PROJECT_ANALYSIS_CN.md` — 项目现状与风险

## 1. 文档目标与范围

### 1.1 目标

1. 在**不破坏现有 Leader/Follower 编队架构**的前提下，增加机间最小安全距离保障；
2. 覆盖两种执行模式：
   - **编队模式（FORMATION）**：长机集中决策，实时修正僚机 GUIDED 目标；
   - **独立模式（INDEPENDENT）**：任务规划阶段空间/时间去冲突 + 运行时监测兜底；
3. 定义可落地的模块接口、算法选型、安全状态机、参数表与测试方案；
4. 预留与 ArduPilot 飞控 OA（Obstacle Avoidance）的融合路径，作为最后一道保险。

### 1.2 范围（In Scope）

| 类别 | 说明 |
|------|------|
| 机间避碰 | 多旋翼/固定翼群控节点间的水平与垂直分离 |
| 预测性避障 | 基于位置 + 速度矢量的 TTC（Time To Collision）预警 |
| 安全降级 | 冲突临界时 LOITER / 扩大编队 / 可选 RTL |
| 独立模式去冲突 | 子任务拆分后的高度分层、时间错开、航带偏移 |
| 可观测性 | 最小间距、TTC、避障修正量、告警等级 |

### 1.3 非范围（Out of Scope，本期不做）

| 类别 | 说明 | 后续方向 |
|------|------|----------|
| 静态障碍物避障 | 建筑物、树木、地形 | 传感器 + ArduPilot OA 或 SLAM |
| 僚机间 P2P 通信 | 分布式 ORCA | Phase 3 可选 |
| 视觉/激光融合 | 深度相机、雷达 | 硬件选型后单独立项 |
| MP 插件 UI | 避障参数可视化面板 | 伴飞稳定后 |

## 2. 现状分析

### 2.1 系统架构（当前）

```
Mission Planner ──(5G/VPN UDP)──► 长机 swarm_node
                                      │
              ┌───────────────────────┼───────────────────────┐
              │ MAVLink UART/UDP      │ UDP:19870 / P900        │
              ▼                       ▼                         ▼
        长机 ArduPilot FC       僚机1 swarm_node           僚机N swarm_node
                                      │                         │
                                      ▼                         ▼
                                僚机1 ArduPilot FC        僚机N ArduPilot FC
```

**控制环**（长机 `control_loop`，默认 10Hz）：

```
navigate_leader()          # 长机航点跟踪，到点半径 3m（多旋翼）
    → formation_.compute() # 几何编队目标
    → air_link_->send(FOLLOWER_CMD)  # 僚机 GUIDED 目标
```

**机间消息**（`LinkMsgType`）：

| 类型 | 方向 | MAVLink 映射 |
|------|------|-------------|
| `STATE_REPORT` | 僚机 → 长机 | `GLOBAL_POSITION_INT` |
| `FOLLOWER_CMD` | 长机 → 僚机 | `SET_POSITION_TARGET_GLOBAL_INT` |

长机通过 `follower_state_cache_` 汇总僚机位置；僚机之间**无直连**。

### 2.2 现有安全机制

| 机制 | 位置 | 说明 |
|------|------|------|
| Guided 保活 | `send_guided_keepalive()` | 防止 `GUID_TIMEOUT` 悬停 |
| 飞控心跳失联 RTL | `check_heartbeat_health()` | 心跳 >5s → RTL |
| 5G 链路降级 | `HealthMonitor` | DEGRADED/LOST → STATUSTEXT/LOITER/RTL |
| 停止编队 | `cmd_stop_formation()` | 切 LOITER |

**缺失**：无机间距离检查、无预测性冲突检测、无避障目标修正。

### 2.3 碰撞风险场景

#### 场景 A：编队转弯交叉

长机变向时，左右翼僚机从两侧向新编队位收敛，若 `delta_dis` 较小或 GPS 误差叠加，**水平路径可能交叉**。

相关代码：`FormationCalculator::compute()` 仅按长机 heading 做几何偏移，未校验僚机间距离。

#### 场景 B：僚机追赶超速

前两个僚机速度 `vel = clamp(0.5 * dis, minV, maxV)`；第 3 架起固定使用长机速度，**无收敛控制**，可能追尾或超越相邻僚机。

#### 场景 C：独立模式共点

`MissionSplitter::split_evenly()` 使相邻子任务在衔接航点处**共享 1 个航点**，各机 AUTO 独立飞行，可能在同一时刻到达同一坐标。

#### 场景 D：高度层不足

默认 `delta_H = 30m`；GPS 垂直误差 ±5~10m 时，名义高度分层可能不足以保证垂直分离。

#### 场景 E：状态上报延迟

`STATE_REPORT` 约 10Hz；5G/P900 弱链路时包龄可达数秒，**基于过时位置的避障可能失效**。

## 3. 设计原则

1. **集中决策、最小侵入**：避障逻辑集中在长机，僚机仍执行 GUIDED 目标，不改变现有角色分工；
2. **分层防御**：规划去冲突（独立模式）→ 实时分离修正（编队）→ 临界 LOITER → 可选 RTL；
3. **可关闭、可降级**：`--avoid-enable 0` 或参数异常时回退纯编队逻辑；
4. **确定性优先**：Phase A 采用可解释的几何分离，便于 SITL 复现与实飞排障；
5. **与飞控职责分离**：伴机负责群协调与目标修正；飞控负责低层跟踪与 OA 反应。

## 5. 核心数据结构

### 5.1 避障参数 `AvoidanceParams`

```cpp
struct AvoidanceParams {
    bool   enable           = true;

    // 水平分离（m）
    double safe_h           = 20.0;   // 警告并开始修正
    double crit_h             = 10.0;   // 临界，触发 LOITER
    double min_formation_dis  = 30.0;   // 编队名义间距下限

    // 垂直分离（m）
    double safe_v           = 15.0;
    double crit_v           = 8.0;

    // 预测窗口
    double t_horizon        = 3.0;    // s，VO/TTC 预测时长
    double t_crit             = 2.0;    // s，TTC 低于此值 → CRITICAL

    // 修正增益
    double sep_gain         = 10.0;   // 分离向量推开增益（m）
    double max_correction_h = 25.0;   // 单帧最大水平修正（m）
    double vel_scale_min    = 0.3;    // 近距离最低速度比例

    // 状态时效
    uint64_t state_max_age_ms = 2000; // 僚机状态超过此龄不参与避障

    // 编队感知
    bool   widen_on_turn    = true;   // 转弯预展宽
    double turn_widen_factor  = 1.3;    // 展宽系数
};
```

### 5.2 避障等级 `AvoidanceLevel`

| 等级 | 条件（满足任一） | 动作 |
|------|------------------|------|
| `NONE` | 所有间距 > safe | 无修正 |
| `WARN` | safe_h ≥ d_h > crit_h 或 TTC < t_horizon | 修正目标 + GCS STATUSTEXT |
| `CRITICAL` | d_h ≤ crit_h 或 TTC ≤ t_crit | 长机 LOITER + 僚机 hold + 告警 |
| `STALE` | 有效机体数 < 2 或状态过期 | 暂停编队扩张，仅保活 |

### 5.3 机体快照 `AvoidanceBody`

```cpp
struct AvoidanceBody {
    int    sysid;
    double lat, lon;       // rad
    double alt_rel;        // m
    double vn, ve, vd;     // m/s NED（需扩展 STATE_REPORT）
    uint64_t state_age_ms;
    bool   valid;
};
```

### 5.4 输出 `AvoidanceResult`

```cpp
struct AvoidanceResult {
    AvoidanceLevel level = AvoidanceLevel::NONE;
    std::vector<FollowerTarget> targets;
    double min_pair_distance_h = 1e9;
    double min_ttc             = 1e9;
    int    pairs_in_conflict   = 0;
    double max_correction_m    = 0;
};
```

## 6. 算法设计

### 6.1 坐标系与距离计算

避障计算统一在**以长机当前位置为原点的局部 ENU 平面**进行（复用 `lla_to_flat_earth()` / `flat_earth_to_lla()`）：

```
x = 北向 (m), y = 东向 (m), z = -alt_rel（向上为正）
d_h = sqrt((x_i-x_j)² + (y_i-y_j)²)
d_v = |z_i - z_j|
```

水平距离采用现有 `distance_between()`；局部 ENU 用于分离向量方向，避免反复三角函数换算。

### 6.2 Phase A：分离约束（首选落地）

**目标**：实现简单、可预测、易调试。

对每个机体对 (i, j)，若 `d_h < safe_h` 且 `d_v < safe_v`：

1. **分离方向**（水平）：  
   `u_sep = normalize(p_i - p_j)`（从 j 指向 i，推开 i）

2. **推开量**（类似人工势场）：  
   `Δp = sep_gain * (1/d_h - 1/safe_h) * u_sep`，限制 `|Δp| ≤ max_correction_h`

3. **速度缩放**：  
   `v' = v * clamp(d_h / safe_h, vel_scale_min, 1.0)`

4. **长机参与**：长机与最近僚机距离不足时，长机航点目标同样后移或减速（可选 Phase A+）。

**伪代码**：

```
for each body i in {leader, followers}:
    correction_i = 0
    for each body j != i:
        if stale(j): continue
        if d_h(i,j) < safe_h and d_v(i,j) < safe_v:
            correction_i += repulsive(i, j)
    apply correction_i to target_i (position + vel scale)
level = max over pairs of level_from(d_h, d_v, ttc)
```

**优点**：实现快、计算量 O(n²)（n ≤ 10 可忽略）。  
**缺点**：多机对称冲突时可能振荡；需 `max_correction_h` 限幅。

### 6.3 Phase B：速度障碍法（VO / 简化 ORCA）

**动机**：分离约束仅看当前距离；两机高速对向时，需预测轨迹。

**相对运动**（机体 i 相对 j）：

```
r = p_j - p_i          # 相对位置（j 在 i 的方位）
v_rel = v_i - v_j      # 相对速度
```

**TTC（仅当接近时有效）**：

```
approach = -dot(r, v_rel)
if approach <= 0: TTC = ∞   # 正在远离
else:
    TTC = |r|² / approach    # 简化线性近似
```

**速度障碍锥**：若存在 `τ ∈ (0, t_horizon]` 使 `|r + v_rel * τ| < R`，则当前速度组合不安全。

**修正策略（简化 VO）**：

1. 在水平面内搜索最小扰动 `Δv`，使修正后速度 `v' = v + Δv` 脱离 VO 锥；
2. 约束 `|Δv| ≤ v_max * 0.5`；
3. 将 `v'` 转换为 GUIDED 目标点的隐含速度（通过 `FOLLOWER_CMD` 的 vel + hdg，或 `SET_POSITION_TARGET` 的 vx/vy）。

**ORCA 扩展（3 机以上推荐）**：

对每对 (i,j) 构造 ORCA 半平面约束，求满足所有约束且最接近期望速度 `v_pref` 的 `v_new`（线性规划，2D 下 n≤10 可解析求解）。

Phase B 在 Phase A 之后叠加：先用 VO 修正速度，再用分离约束修正位置。

### 6.4 Phase C：编队感知策略

| 策略 | 触发 | 动作 |
|------|------|------|
| 转弯预展宽 | 长机 heading 变化率 > 阈值（如 5°/s） | 临时 `delta_dis *= turn_widen_factor` |
| 冲突时扩编 | `level == WARN` 持续 > 2s | `delta_dis += 10m`，直至 `min_formation_dis` 上限 |
| 长机减速 | 最小 TTC < t_horizon | 长机目标速度 × 0.7 |
| 僚机 hold | `level == CRITICAL` | 僚机目标设为当前位置（保活） |

### 6.5 独立模式：任务规划去冲突

在 `MissionSplitter` 输出后、`TASK_*` 下发前执行 **MissionDeconflict**：

#### 策略 1：高度分层（默认开启）

```
僚机 i 的所有航点：alt += i * layer_alt_step   (默认 layer_alt_step = 20m)
约束：alt ∈ [minH, maxH]
```

#### 策略 2：时间错开

```
僚机 i 的 TASK_START 延迟：start_delay_ms = i * stagger_ms  (默认 5000ms)
```

#### 策略 3：航带平行偏移

沿主航线法向偏移：

```
offset_m = (i - (n-1)/2) * corridor_spacing   (默认 30m)
每个航点 lat/lon 做法向 offset_position()
```

#### 策略 4：共点消解

对 `MissionSplitter` 产生的**共享衔接航点**：

- 为每机生成**略微错开**的替代点（沿航线前后 ±5m），或
- 仅允许 sequential 通过（前机离开后后机再 START）

**推荐组合**：高度分层 + 时间错开（实现简单、AUTO 兼容性好）。

运行时兜底：长机仍汇总 `STATE_REPORT`，若独立模式下检测到 `d_h < crit_h`，经 `CTRL_CMD` 代理发送 LOITER 给相关僚机。

## 7. 协议与状态扩展

### 7.1 STATE_REPORT 速度矢量

当前 `link_mavlink_codec.cpp` 将 `vel` + `hdg` 合成 `vx/vy`，**丢失垂直速度与真实 NED 分量**。

**改进**（向后兼容）：

| 字段 | 来源 | 说明 |
|------|------|------|
| `vn, ve, vd` | 僚机 `UnitState` | 写入 `GLOBAL_POSITION_INT.vx/vy/vz`（cm/s） |
| `vel` | 保留 | 水平速度模，供旧逻辑 |
| 协议标记 | `gp.alt == 1002` | 表示 v2 含真实 NED 速度 |

僚机 `run_follower_tick()` 填充：

```cpp
pkt.vn = st.vn;  // 新增 LinkPacket 字段
pkt.ve = st.ve;
pkt.vd = st.vd;
```

### 7.2 可选：CONFLICT_WARNING 消息

Phase C 可选新增 `LinkMsgType::CONFLICT_WARNING`（长机 → 全体僚机），映射 `STATUSTEXT` 或自定义 `DATA96`，携带：

- `min_distance`, `ttc`, `conflict_sysid_a`, `conflict_sysid_b`

本期可不实现专用消息，优先用 GCS `STATUSTEXT` + 本地日志。

### 7.3 GCS 可配置参数（建议）

通过 `COMMAND_LONG` 扩展或 MP 参数页：

## 8. 安全状态机

### 8.1 状态定义

```
                    ┌──────────┐
         启用避障   │  NORMAL  │◄────────────────┐
        ──────────► │  正常编队 │                 │
                    └────┬─────┘                 │
                         │ d < safe_h            │ d > safe_h 持续 3s
                         ▼                       │
                    ┌──────────┐                 │
                    │   WARN   │─────────────────┘
                    │ 修正目标  │
                    └────┬─────┘
                         │ d < crit_h 或 TTC < t_crit
                         ▼
                    ┌──────────┐
                    │ CRITICAL │
                    │ LOITER   │
                    └────┬─────┘
                         │ 人工干预 / RTL 命令 / 超时 60s
                         ▼
                    ┌──────────┐
                    │  RECOVER │ → RTL 或 等待 GCS START_FORMATION
                    └──────────┘
```

### 8.2 CRITICAL 动作序列

1. 长机：`fc->set_mode(LOITER)`，停止 `navigate_leader` 航点推进；
2. 僚机：经 `FOLLOWER_CMD` 下发**当前位置**为 hold 点；
3. GCS：`STATUSTEXT`「COLLISION CRITICAL: min_d=xx m, pair=A-B」；
4. 停止扩大编队与任务推进，直至 GCS 发送 `START_FORMATION` 或 `STOP_FORMATION`；
5. 可选：60s 无人工确认 → 全体 RTL（可配置 `--avoid-auto-rtl`）。

### 8.3 与现有安全机制关系

| 现有机制 | 关系 |
|----------|------|
| 飞控心跳 RTL | 独立并行，避障不替代 |
| 5G LOST → LOITER | 链路 LOST 优先于避障恢复 |
| Guided 保活 | CRITICAL hold 期间必须保持保活 |

## 9. ArduPilot 飞控层融合（可选增强）

### 9.1 职责划分

| 层级 | 职责 |
|------|------|
| 伴机 swarm_node | 群协调、目标修正、机间 TTC |
| ArduPilot OA | 单机遇障物减速/停止（含注入的「虚拟障碍」） |

### 9.2 虚拟障碍物注入

长机或各僚机伴机可向本机飞控发送 `OBSTACLE_DISTANCE`（或 `OBSTACLE_DISTANCE_3D`），将**其他无人机位置**映射为距离扇区，供 OA 使用。

**注意**：

- 需启用飞控 `AVOID_ENABLE` 等参数；
- 更新频率与 GPS 精度需匹配；
- 与伴机 GUIDED 目标修正**可能打架**，建议 OA 仅作 CRITICAL 兜底或独立模式使用。

### 9.3 推荐飞控参数（多旋翼参考）

```
AVOID_ENABLE = 1
AVOID_MARGIN = 5          # m，与伴机 crit_h 协调
PRX1_TYPE = ...           # 按实际传感器
GUID_TIMEOUT = 3
FS_GCS_ENABLE = 1
```

|------|---------|------|

## 14. 风险与对策

| 风险 | 影响 | 对策 |
|------|------|------|
| GPS 误差 1–3m | 虚假 WARN 或漏检 | 增大 safe_h；速度修正为主、位置修正为辅 |
| 机间链路延迟 | 基于过时状态 | `state_max_age_ms` 丢弃；STALE 降级 |
| 修正振荡 | 编队不稳定 | `max_correction_h` 限幅；低通滤波修正量 |
| 与 GUIDED 冲突 | 飞控跟踪滞后 | 限制单帧修正；提高 ctrl_hz |
| P900 带宽不足 | 状态丢包 | 优先保证 STATE_REPORT；降低 synthetic telem |
| 多机 ORCA 计算 | CPU 占用 | n≤10 时 2D LP 可忽略；否则仅 Phase A |
